API reference: The fatiando package¶

The fatiando package contains subpackages for different geophysical methods and other utilities.

See the API reference for each subpackage for a list of all functions and classes defined by it.

fatiando.gridder: Grids and irregularly spaced data¶

Create and operate on data grids, scatters, and profiles.

regular(area, shape[, z]) Create a regular grid.
scatter(area, n[, z, seed]) Create an irregular grid with a random scattering of points.
circular_scatter(area, n[, z, random, seed]) Generate a set of n points positioned in a circular array.
cut(x, y, scalars, area) Return a subsection of a grid.
inside(x, y, area) Tell which indices of an array fall inside an area.
profile(x, y, v, point1, point2, size[, …]) Extract a profile between 2 points from spacial data.
interp(x, y, v, shape[, area, algorithm, …]) Interpolate spacial data onto a regular grid.
interp_at(x, y, v, xp, yp[, algorithm, …]) Interpolate spacial data onto specified points.
spacing(area, shape) Returns the spacing between grid nodes
pad_array(a[, npd, padtype]) Return a padded array of arbitrary dimension.
unpad_array(a, nps) Remove padding from an array.
pad_coords(xy, shape, nps) Apply padding to coordinate vectors.

fatiando.seismic: Seismology and seismics¶

Tools for seismic and seismology, like convolutional modeling and wave propagation.

Forward modeling and inversion¶

  • ttime2d: 2D seismic ray travel-time modeling
  • epic2d: 2D epicenter determination
  • profile: Modeling and inversion of seismic profiling
  • wavefd: Finite difference solution of the 2D elastic wave equation
  • conv: Convolutional seismic modeling

Tomography¶

  • srtomo: 2D straight-ray tomography

Classes¶

RickerWavelet(f[, amp, delay]) The Ricker (mexican hat) wavelet.

Functions¶

lame_mu(vs, density) Calculate Lamé’s second parameter \(\mu\) (the shear modulus).
lame_lambda(vp, vs, density) Calculate Lamé’s first parameter \(\lambda\).

fatiando.gravmag: Gravity and magnetics¶

Gravity and magnetics forward modeling, inversion, transformations and utilities.

Inversion¶

The inversion modules use the forward modeling models and the fatiando.inversion package to solve potential field inverse problems:

  • basin2d: 2D inversion of the shape of sedimentary basins and other outcropping bodies
  • harvester: 3D inversion of compact bodies by planting anomalous densities
  • euler: 3D Euler deconvolution methods to estimate source location
  • magdir: Inversion methods to estimate the total magnetization vector of multiple sources.

Processing¶

The processing modules offer tools to prepare potential field data before or after modeling.

  • normal_gravity: Compute normal gravity and reductions.
  • eqlayer: Equivalent layer processing
  • transform: Potential field transformations, like upward continuation, derivatives, etc
  • imaging: Imaging methods for potential fields for estimating physical property distributions
  • tensor: Utilities for operating on the gradient tensor

Interactivity¶

Module interactive implements matplotlib GUIs and IPython HTML widgets for interacting with the modeling and processing functions.

Forward modeling¶

The fatiando.gravmag package defines a few modules that calculate the potential fields of some geometric primitives.
